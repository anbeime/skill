## 小红书平台特点

### 平台定位
- **类型**：图文社区
- **用户群体**：年轻女性为主，追求生活品质
- **内容风格**：种草、测评、生活方式、干货分享
- **视觉要求**：图片精美，排版舒适

### 内容特点
- **长度**：300-800字
- **格式**：纯文本+emoji
- **排版**：短段落，每段不超过2行
- **标题**：15-20字，吸睛短标题+emoji
- **封面**：3:4竖屏，建议1080x1440

## 内容创作规范

### 标题策略

**格式要求**：
- 不超过20字
- 包含emoji增加吸引力
- 吸引点击的技巧

**常用标题模板**：
- 感叹型：「AI太香了！🤖」
- 疑问型：「你准备好了吗？⚡」
- 数字型：「这3点你必须知道！💡」
- 对比型：「用vs不用，差距太大了！」
- 秘密型：「终于发现了！✨」

**示例**：
```
✅ AI时代来了！你准备好了吗？🤖
✅ 5个效率神器让工作效率翻倍🚀
✅ 终于发现了！这样学AI最快💡
✅ 用了vs没用，差距太大了！😱
```

### 正文结构

**标准结构**：
```
标题：吸睛短标题（15-20字，含emoji）

开场白：引发共鸣

📌 核心要点：
✅ 要点一
✅ 要点二
✅ 要点三

💡 建议/技巧：
1. 建议1
2. 建议2
3. 建议3

结尾互动 + 标签
```

**排版技巧**：
- 短段落清晰，每段不超过2行
- 使用emoji增加视觉吸引力
- 使用符号标注（✅、💡、📌）
- 结尾添加相关标签

**emoji使用规范**：
- 每段1-2个即可，不要过度
- 使用相关emoji，不要随意堆砌
- 常用emoji：✅、💡、📌、🚀、🤖、⚡、✨、🔥、💯

## 图片卡片生成

### 为什么要生成图片卡片？

小红书的视觉效果非常重要，精美的图片卡片可以：
- 提升点击率
- 增强内容记忆点
- 提升专业度
- 便于传播

### 图片规格

**封面卡片**：
- 尺寸比例：3:4
- 基准尺寸：1080×1440px
- 包含：Emoji装饰、大标题、副标题
- 样式：渐变背景+圆角内容区

**正文卡片**：
- 尺寸比例：3:4
- 基准尺寸：1080×1440px
- 支持：标题、段落、列表、引用、代码块、图片
- 样式：白色卡片+渐变背景边框

### 生成方式

**方式一：使用Auto-RedNote.skill的渲染脚本**

如果您有Auto-RedNote.skill，可以使用它的渲染脚本：

```bash
python scripts/render_xhs.py content.md
```

**Markdown文档格式**：
```markdown
---
emoji: "💡"
title: "5个效率神器让工作效率翻倍"
subtitle: "对着抄作业就好了"
---

# 📝 神器一：Notion

全能型笔记工具，支持数据库、看板、日历等多种视图...

## 特色功能

- 特色一
- 特色二

# ⚡ 神器二：Raycast

Mac必备启动器，效率提升神器...

## 推荐原因

- 原因一
- 原因二

# 🌈 神器三：Arc

全新理念的浏览器，侧边栏标签管理...

```

**渲染参数**：
```bash
# 默认主题 + 手动分隔分页
python scripts/render_xhs.py content.md -m separator

# 固定1080x1440，自动缩放文字
python scripts/render_xhs.py content.md -m auto-fit

# 自动切分分页（推荐）
python scripts/render_xhs.py content.md -m auto-split

# 动态高度
python scripts/render_xhs.py content.md -m dynamic --max-height 4320

# 切换主题
python scripts/render_xhs.py content.md -t playful-geometric -m auto-split
```

**排版主题**：
- `default`：默认简约浅灰渐变背景
- `playful-geometric`：活泼几何（Memphis）
- `neo-brutalism`：新粗野主义
- `botanical`：植物园自然
- `professional`：专业商务
- `retro`：复古怀旧
- `terminal`：终端命令行
- `sketch`：手绘素描

## 小红书发布

### 发布接口

**接口路径**：`POST /publish-xiaohongshu`

**认证方式**：Cookie

**请求格式**：
```json
{
  "title": "笔记标题",
  "content": "笔记内容",
  "cover": "封面图URL",
  "digest": "摘要/标签"
}
```

**使用方式**：
```bash
python scripts/publish.py --platform xiaohongshu \
  --title "AI时代来了！你准备好了吗？🤖" \
  --content "最近玩了很多AI工具，真的太香了！✨\n\n📌 核心观点：\n✅ AI提升效率..." \
  --cover-image "https://example.com/cover.png" \
  --tags "AI,黑科技,效率工具"
```

### 发布到小红书（使用Auto-RedNote.skill）

如果您有Auto-RedNote.skill，可以使用它的发布脚本：

```bash
python scripts/publish_xhs.py \
  --title "笔记标题" \
  --desc "笔记描述" \
  --images card_1.png card_2.png cover.png
```

**前置条件**：
需配置小红书Cookie（已自动创建.env文件）：
```
XHS_COOKIE=your_cookie_string_here
```

**Cookie获取方式**：
1. 在浏览器中登录小红书（https://www.xiaohongshu.com）
2. 打开开发者工具（F12）
3. 在Network标签中查看请求头的Cookie
4. 复制完整的cookie字符串并填入.env文件

## 常见问题

### Q1：小红书笔记字数有限制吗？
A：建议300-800字，太长影响阅读体验。

### Q2：emoji可以用多少？
A：适度使用，每段1-2个即可，不要过度。

### Q3：标签需要加#吗？
A：内容中不需要，发布时智能体会自动添加。

### Q4：图片必须用卡片吗？
A：不是必须，但精美的图片卡片可以提升点击率。

### Q5：如何提高笔记曝光？
A：
- 精美的封面
- 吸睛的标题
- 有价值的内容
- 合理的标签
- 适时的发布时间
