# 产品功能评审会议模板

## 会议类型
产品决策会议 - 功能优先级评审

## 会议目标
评审产品功能的优先级，决定下一季度的开发计划，确保资源投入到最有价值的功能上。

## 推荐智能体角色

### 必选角色
1. **产品经理** (主持人)
   - 职责: 主持会议、展示RICE排序、协调讨论
   - 专业工具: RICE优先级排序脚本
   - 发言重点: 用户价值、产品战略

2. **技术架构师**
   - 职责: 评估技术可行性和实现难度
   - 发言重点: 技术风险、架构影响、开发成本

3. **市场分析师**
   - 职责: 评估市场价值和竞争态势
   - 发言重点: 市场需求、竞品对比、用户反馈

### 可选角色
4. **财务顾问**
   - 职责: 评估成本收益和ROI
   - 发言重点: 投入产出比、资源效率

5. **项目经理**
   - 职责: 评估资源和时间
   - 发言重点: 团队容量、交付时间、依赖关系

## 会议流程

### 阶段一: 数据准备 (会前)
产品经理智能体执行：
```bash
# 准备功能列表CSV文件
# 格式: name,reach,impact,confidence,effort
python scripts/rice_prioritizer.py features.csv --capacity 15
```

输出：
- RICE排序结果
- 快速胜利 vs 大赌注分析
- 工作量分布

### 阶段二: 开场陈述 (5分钟)
1. 产品经理介绍会议目标和背景
2. 展示RICE排序结果
3. 说明评审标准和决策依据

### 阶段三: 逐项评审 (20分钟)
针对每个功能：
1. 产品经理: 说明功能价值和RICE评分
2. 技术架构师: 评估实现难度和技术风险
3. 市场分析师: 评估市场需求和竞争优势
4. 财务顾问: 评估ROI和资源效率

### 阶段四: 争议讨论 (10分钟)
针对评分差异较大的功能：
- 各角色提供详细论据
- 讨论是否调整RICE评分
- 识别隐藏的风险和机会

### 阶段五: 共识收敛 (5分钟)
1. 总结各方观点
2. 确定最终优先级排序
3. 明确Q2/Q3开发计划

### 阶段六: 决策输出
生成包含以下内容的会议记录：
- 最终优先级排序（含RICE分数）
- 每个功能的评估理由
- 技术风险和注意事项
- 资源分配和时间计划
- 下一步行动项

## 输出格式

```markdown

**会议时间**: [自动生成]
**参与智能体**: 产品经理、技术架构师、市场分析师、财务顾问
**会议目标**: 评审Q2功能优先级

## 最终优先级排序

| 排名 | 功能名称 | RICE分数 | Reach | Impact | Confidence | Effort |
|------|---------|----------|-------|--------|------------|--------|
| 1 | AI自动生成脚本 | 85 | 1000 | 3x | 100% | 3人月 |
| 2 | 多平台发布 | 72 | 800 | 2x | 80% | 2人月 |
| ... | ... | ... | ... | ... | ... | ... |

## 决策理由

### AI自动生成脚本 (优先级: P0)
- **产品经理**: 核心价值功能，用户需求强烈
- **技术架构师**: 技术可行，可复用现有AI模型
- **市场分析师**: 竞品尚未实现，有先发优势
- **财务顾问**: 预期ROI高，3个月回本

### 多平台发布 (优先级: P1)
- **产品经理**: 用户呼声最高的功能
- **技术架构师**: 实现难度中等，需要2人月
- **市场分析师**: 可扩大用户覆盖面
- **财务顾问**: 边际成本低，长期价值高

## 风险和注意事项

### 技术风险
- AI模型需要持续优化，可能影响用户体验
- 多平台发布需要维护多套代码，增加维护成本

### 市场风险
- 竞品可能快速跟进AI功能
- 用户对多平台的实际需求需要验证

## 资源分配计划

**Q2 (4月-6月)**:
- AI自动生成脚本: 3人月 (2名工程师 × 1.5个月)
- 多平台发布: 2人月 (1名工程师 × 2个月)
- 预留缓冲: 1人月

**Q3 (7月-9月)**:
- API开放平台: 4人月
- 用户协作功能: 3人月

- [ ] 项目经理: 制定详细的开发计划 (截止: 4月12日)
- [ ] 市场分析师: 准备功能发布的市场推广方案 (截止: 4月15日)
```

## 使用示例

### 示例1: 标准功能评审
**用户输入**:
```
请用产品团队评审以下功能的优先级：
1. AI自动生成脚本
2. 多平台发布
3. 数据分析看板
4. 用户协作功能
5. API开放平台
```

**系统执行**:
1. 产品经理智能体调用RICE脚本生成初步排序
2. 召开会议，各智能体发言讨论
3. 输出完整的会议记录和优先级列表

### 示例2: 带约束条件的评审
**用户输入**:
```
评审Q2功能优先级，团队容量15人月，必须包含1个战略功能
```

**系统执行**:
1. RICE排序时考虑15人月容量约束
2. 会议讨论时优先考虑战略功能
3. 输出符合约束条件的路线图

## 工具支持

### RICE优先级排序脚本
```bash
# 基础用法
python scripts/rice_prioritizer.py features.csv

# 指定团队容量
python scripts/rice_prioritizer.py features.csv --capacity 15

# JSON输出（便于集成）
python scripts/rice_prioritizer.py features.csv --output json
```

### CSV文件格式
```csv
name,reach,impact,confidence,effort
AI自动生成脚本,1000,massive,high,m
多平台发布,800,high,medium,s
数据分析看板,500,medium,high,l
用户协作功能,600,high,low,m
API开放平台,400,massive,medium,xl
```

**字段说明**:
- `reach`: 每季度影响的用户数
- `impact`: massive/high/medium/low/minimal
- `confidence`: high/medium/low
- `effort`: xl/l/m/s/xs (超大/大/中/小/超小)

## 注意事项

1. **数据准备很重要**: 会前必须准备好功能列表和RICE评分
2. **保持讨论聚焦**: 避免陷入技术细节，专注于优先级决策
3. **记录争议点**: 对于有争议的功能，详细记录各方观点
4. **明确下一步**: 会议结束时必须有明确的行动项和责任人
5. **定期回顾**: 每季度回顾优先级决策的准确性，持续优化
