# 客户访谈洞察分析会议模板

## 会议类型
产品研究会议 - 客户洞察分析

## 会议目标
分析客户访谈记录，提取关键洞察，生成产品改进方案和PRD草稿。

## 推荐智能体角色

1. **产品经理** (主持人)
   - 职责: 主持会议、解读洞察、制定改进方案
   
2. **用户研究员**
   - 职责: 分析访谈记录、提取痛点和需求
   - 专业工具: customer_interview_analyzer.py
   
3. **设计师**
   - 职责: 从体验角度提出改进方向
   
4. **技术架构师**
   - 职责: 评估解决方案的技术可行性

## 会议流程

### 阶段一: 数据分析 (会前)
用户研究员智能体执行：
```bash
python scripts/customer_interview_analyzer.py interview_transcript.txt
```

输出：
- 痛点列表（含严重程度）
- 功能需求（含优先级）
- Jobs-to-be-Done
- 情感分析
- 关键主题和引用

### 阶段二: 洞察展示 (10分钟)
用户研究员：
1. 展示分析结果
2. 解释关键痛点
3. 说明用户情感和态度

### 阶段三: 深度讨论 (20分钟)
1. 产品经理: 从产品战略角度解读洞察
2. 设计师: 提出体验改进方向
3. 技术架构师: 评估技术可行性
4. 用户研究员: 补充用户行为细节

### 阶段四: 方案生成 (15分钟)
1. 确定优先级最高的3个痛点
2. 为每个痛点设计解决方案
3. 评估方案的价值和难度

### 阶段五: PRD草稿 (10分钟)
产品经理使用PRD模板生成需求文档草稿

## 改进方案

### 方案1: AI自动生成脚本
**解决的痛点**: 脚本生成速度慢、不知道如何写
**方案描述**: 
- 用户输入关键词或主题
- AI自动生成3个不同风格的脚本
- 用户可以选择和编辑

**技术可行性**: 高（可使用现有大模型）
**预期价值**: 提升50%的生成效率
**实现周期**: 3人月

### 方案2: 批量处理功能
**解决的痛点**: 无法批量生成
**方案描述**:
- 支持上传CSV文件
- 批量生成脚本
- 导出为ZIP文件

**技术可行性**: 高
**预期价值**: 重度用户满意度提升30%
**实现周期**: 1人月

## PRD草稿

**问题陈述**:

1. 不知道如何写出吸引人的脚本
2. 手动编写耗时长，效率低

**解决方案**:
提供AI自动生成功能，用户输入关键词，系统生成3个不同风格的脚本供选择。

**成功指标**:
- 脚本生成时间从5分钟降低到1分钟
- 用户满意度提升至4.5/5
- 功能使用率达到60%

- 作为新手用户，我希望有AI辅助，这样我可以学习如何写好脚本

**验收标准**:
- [ ] 用户可以输入关键词

- [ ] 生成时间 < 30秒

## 使用示例

### 示例: 分析用户访谈并生成改进方案

**用户输入**:
```
分析这份客户访谈记录并生成产品改进方案：

访谈对象：张三，内容创作者，使用产品3个月
访谈时间：2024-04-01

Q: 你在使用产品时遇到的最大问题是什么？
A: 每次生成脚本都要花很长时间想怎么写，太浪费时间了。有时候想不出来，就只能放弃。

Q: 你希望产品有什么改进？
A: 如果能有AI自动生成就好了，我输入主题，它自动帮我生成几个脚本，我选一个修改就行。

Q: 你愿意为这个功能付费吗？
A: 当然愿意，如果能节省我的时间，付费是值得的。
```

**系统执行**:
1. 用户研究员调用分析脚本提取洞察
2. 召开会议，各智能体讨论改进方案
3. 输出包含洞察、方案和PRD草稿的完整报告

## 注意事项

1. **访谈记录要完整**: 包含用户原话，便于提取关键引用
2. **关注用户行为**: 重点分析用户实际行为，而非意图
3. **挖掘深层需求**: 通过"为什么"挖掘用户的真实需求
4. **验证方案**: 生成方案后，建议继续访谈验证
5. **快速迭代**: PRD草稿应快速生成，后续持续迭代优化
