# 项目风险评估会议模板

## 会议类型
项目风险评估会议

## 适用场景
- 新项目立项评估
- 项目阶段性风险评估
- 项目变更影响评估
- 项目终止决策
- 项目优先级排序

## 推荐角色组合

### 最小组合（4人）
1. **项目经理**: 主持讨论，从项目管理角度评估
2. **技术专家**: 评估技术可行性风险
3. **业务负责人**: 评估业务价值和市场风险
4. **法务顾问**: 评估合规和法律风险

### 完整组合（6人）
1. **项目经理**: 进度、资源、风险管控
2. **技术专家**: 技术实现和架构风险
3. **运营专家**: 用户和运营风险
4. **法务顾问**: 合规和知识产权风险
5. **财务顾问**: 成本和预算风险
6. **部门负责人**: 资源支持和战略决策

## 典型讨论点

### 技术风险
- 技术选型和架构风险
- 团队能力和经验不足
- 第三方依赖风险
- 技术债务积累

### 运营风险
- 用户接受度和市场风险
- 运营能力和资源不足
- 数据安全和隐私风险
- 监管政策变化

### 项目风险
- 时间延期风险
- 成本超支风险
- 人员流失风险
- 需求变更频繁

### 商业风险
- 竞争对手动作
- 市场需求变化
- 收入不达预期
- 投资回报率低

## 风险评估维度

| 维度 | 评估标准 |
|------|---------|
| **可能性** | 极低/低/中/高/极高 |
| **影响程度** | 极小/小/中/大/极大 |
| **可发现性** | 易于发现/难以及时发现 |
| **可控性** | 完全可控/部分可控/难以控制 |

## 典型决策结果类型

- **绿灯**: 风险可控，建议启动项目
- **黄灯**: 需制定风险缓解措施后方可启动
- **红灯**: 风险过高，建议暂缓或取消
- **调整**: 需要修改项目范围或目标后再评估
- **分阶段**: 分阶段实施，降低整体风险

## 示例会议主题
- 新项目立项的风险评估
- 核心系统重构项目可行性评估
- 海外市场拓展项目风险评估
- 大型技术平台建设项目风险评估
- AI产品开发项目风险评估
